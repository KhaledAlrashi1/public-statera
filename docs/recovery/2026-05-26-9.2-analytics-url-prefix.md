## What was found

12 of 13 `analyticsApi` methods in `apps/web/src/lib/api.ts` were calling Flask-era bare
`/api/<name>` paths. Hono mounts both analytics routers at `/api/analytics`, so every
route is at `/api/analytics/<name>`. Every mismatch produced HTTP 404 from the moment the
frontend first deployed in commit 27ae763 (2026-05-24).

The one correct entry — `accountOverview` at `/api/analytics/account-overview` — was
written correctly in the same initial commit (`c925cc3`) that introduced all 12 wrong paths.
The correct URL convention was known at copy time but not applied consistently.

**User-visible impact before fix:**
- Dashboard: all numeric values / charts empty or zero-state
- Budget page: budget metrics empty
- Insights page: charts unpopulated, no weekly digest
- Income page: income history empty
- Expenses page: category trend chart empty
- Profile page: income detection returned nothing
- SpendingIntelligencePage: 404 (see "Deferred" section below)

Real-user impact: **zero** — site was unannounced throughout the window (2026-05-24 to
2026-05-26).

## What was fixed

Single commit. One file changed: `apps/web/src/lib/api.ts`.

12 URL strings in the `analyticsApi` object (lines 681–767) updated from `/api/<name>` to
`/api/analytics/<name>`:

| Method | Old URL | New URL |
|---|---|---|
| `spendByCategory` | `/api/spend-by-category` | `/api/analytics/spend-by-category` |
| `spendByMonth` | `/api/spend-by-month` | `/api/analytics/spend-by-month` |
| `dashboardMetrics` | `/api/dashboard-metrics` | `/api/analytics/dashboard-metrics` |
| `budgetMetrics` | `/api/budget-metrics` | `/api/analytics/budget-metrics` |
| `safeToSpend` | `/api/safe-to-spend` | `/api/analytics/safe-to-spend` |
| `dashboardBundle` | `/api/dashboard-bundle` | `/api/analytics/dashboard-bundle` |
| `incomePattern` | `/api/income-pattern` | `/api/analytics/income-pattern` |
| `expenseBreakdown` | `/api/expense-breakdown` | `/api/analytics/expense-breakdown` |

## What was deliberately deferred

### 1. `spendingIntelligence` — missing Hono route

`analyticsApi.spendingIntelligence` calls `/api/spending-intelligence`. This is a Flask-era
composite endpoint (`top_merchants`, `category_benchmarks`, `category_deltas`,
`recurring_bills`) that was never ported to Hono. No route exists at any path in the Hono
backend for this data.

The SpendingIntelligencePage nav link is already commented out in `AppShell.tsx:45`. The
page is still reachable via direct URL and the command palette.

**Decision needed (separate 9.x item):** build the Hono composite route OR formally defer
and hide the page. The data it aggregates already exists in other Hono endpoints (R1, R5,
R6, R12 cover the underlying data); assembly into a `/api/analytics/spending-intelligence`
route is the likely path. Not scoped for this commit.

### 2. Network-level test mocking

The unit test suite mocks at the `analyticsApi` function level. No test asserts that
`fetch` was called with a specific URL. This is why the 12-of-13 broken-URL state was
invisible to the test suite from 2026-05-24 until this manual detection.

Tracked as `TODO(module-9-network-mocking)` in CLAUDE.md (8e module entry). MSW (Mock
Service Worker) is the likely addition. Not scoped for any current Module 9 sub-item.

## Post-deploy smoke checklist

After CI deploys this commit, verify each of the following pages on the **live site**
at https://staterafinance.app. Log in with your normal account.

For each page: **numeric data present? Y/N + brief note**

1. **Dashboard** — KWD amounts visible (not KD 0.000), income/expense chart rendered,
   safe-to-spend card shows a number. If zero-state: note which card(s) are empty.

2. **Budget** (`/plan`) — Budget metrics (budget utilisation, spend vs. budget) show
   real numbers, not blank or zero. If zero-state: note the section.

3. **Insights** (`/insights`) — Charts populated (monthly spend, income/expense trend),
   weekly digest section shows content. If empty: note which section.

4. **Income** (`/income`) — Income history chart shows data points (if transactions
   exist), income detection populates. If empty: note the component.

5. **Expenses** (`/expenses`) — Category breakdown chart populated. If empty: note.

6. **Profile** (`/profile`) — Income detection section shows a detected amount or
   "no pattern detected" (not a blank/loading state). If blank: note.

If **all six are green**: 9.2 is closed. Report commit SHA and confirm in session.

If **any page still shows zero-state**: STOP and flag. This indicates a response-shape
bug beneath the URL bug (e.g., the frontend expects `{ items: [...] }` but Hono returns
`{ ok: true, data: [...] }`, or vice versa). Each such case is a separate Module 9
sub-item requiring a scoping grep + proposal cycle.
